Terminal · Git-Workflow · GitHub · VS Code Shortcuts · Einsteiger-Referenz 2025
VS CODE & GIT – EINFÜHRUNG
  • Git speichert jeden Änderungsstand deines Projekts als sogenannten "Commit".
  • GitHub ist die Online-Plattform, auf der du dein Repo teilst und mit anderen zusammenarbeitest.
  • VS Code hat ein eingebautes Terminal (Strg+`), das du direkt im Editor nutzen kannst.
  • Einstieg: git init → git add . → git commit -m"Start" → git push
Befehlsreferenz
GIT-WORKFLOW – GRUNDBEFEHLE
  • git init → Neues lokales Repository anlegen
  • git clone [URL] → Bestehendes Repo herunterladen
  • git status → Geänderte Dateien anzeigen
  • git add . → Alle Änderungen für Commit vormerken
  • git add [Datei] → Einzelne Datei vormerken
  • git commit -m "Text" → Änderungen mit Nachricht speichern
  • git push → Commits ins Remote-Repo hochladen
  • git pull → Aktuelle Änderungen vom Remote holen
  • git fetch → Remote-Stand abrufen (ohne Merge)
  • git log --oneline → Commit-Verlauf kompakt anzeigen
  • git diff → Ungespeicherte Änderungen vergleichen
  • git diff HEAD~1 → Vergleich mit letztem Commit
BRANCH & MERGE
  • git branch → Alle lokalen Branches anzeigen
  • git branch [Name] → Neuen Branch erstellen
  • git switch [Name] → Zu einem Branch wechseln
  • git switch -c [Name] → Branch erstellen & direkt wechseln
  • git merge [Branch] → Branch in aktuellen zusammenführen
  • git rebase [Basis] → Commits auf neue Basis setzen
  • git cherry-pick [Hash] → Einzelnen Commit übernehmen
  • git branch -d [Name] → Lokalen Branch löschen
  • git branch -D [Name] → Branch erzwungen löschen
  • git stash → Änderungen zwischenspeichern
  • git stash pop → Zwischenspeicher wiederherstellen
  • git stash list → Alle Stashes anzeigen
TERMINAL-NAVIGATION
  • pwd → Aktuellen Pfad anzeigen
  • ls / dir → Ordnerinhalt anzeigen (Linux / Windows)
  • ls -la → Alle Dateien inkl. versteckter anzeigen
  • cd [Ordner] → In Ordner wechseln
  • cd .. → Einen Ordner zurück
  • cd ~ → Ins Home-Verzeichnis wechseln
  • mkdir [Name] → Neuen Ordner erstellen
  • touch [Datei] → Neue leere Datei erstellen
  • rm [Datei] → Datei löschen
  • rm -rf [Ordner] → Ordner rekursiv löschen (Vorsicht!)
  • cp [Quelle] [Ziel] → Datei oder Ordner kopieren
  • mv [Alt] [Neu] → Datei verschieben oder umbenennen
  • cat [Datei] → Dateiinhalt im Terminal ausgeben
  • echo "Text" > datei.txt → Text in Datei schreiben
GITHUB REMOTE & CLI
  • git remote -v → Verknüpfte Remote-URLs anzeigen
  • git remote add origin [URL] → Remote-Repo verknüpfen
  • git push -u origin main → Ersten Push inkl. Tracking
  • git push --tags → Alle lokalen Tags hochladen
  • git tag v1.0.0 → Version taggen
  • gh auth login → GitHub CLI: einloggen
  • gh repo clone [Name] → Repo per GitHub CLI klonen
  • gh pr create → Pull Request über CLI erstellen
  • gh pr list → Offene Pull Requests anzeigen
  • gh pr merge → Pull Request zusammenführen
  • gh issue create → Neues Issue erstellen
  • gh release create v1.0 → Release erstellen und veröffentlichen
VS CODE TERMINAL – SHORTCUTS
  • Strg + ` → Terminal öffnen / schließen
  • Strg + Shift + ` → Neues Terminal-Tab öffnen
  • Strg + Shift + 5 → Terminal horizontal teilen
  • Alt + ↑ / ↓ → Zwischen Terminal-Tabs wechseln
  • Strg + C → Laufenden Prozess abbrechen
  • ↑ / ↓ → Befehlshistorie durchblättern
  • Tab → Autovervollständigung
  • clear / cls → Terminal-Ausgabe leeren
  • Strg + Shift + P → Befehlspalette von VS Code öffnen
  • code . → VS Code im aktuellen Ordner öffnen
  • Strg + K, Strg + O → Ordner in VS Code öffnen
  • Strg + Shift + G → Git-Ansicht in VS Code öffnen
GIT RÜCKGÄNGIG & KORRIGIEREN
  • git reset HEAD~1 → Letzten Commit rückgängig (Änder. bleiben)
  • git reset --hard HEAD~1 → Letzten Commit + Änderungen verwerfen
  • git revert [Hash] → Commit durch neuen Gegenzug aufheben
  • git commit --amend → Letzte Commit-Nachricht korrigieren
  • git restore [Datei] → Datei auf Stand des letzten Commits zurück
  • git restore --staged [] → Datei aus dem Stage-Bereich entfernen
  • git clean -fd → Ungetrackte Dateien und Ordner löschen
MODIFIER & OPTIONEN
  • --global → Einstellung für alle Repos setzen
  • --cached → Nur den Stage-Bereich bearbeiten
  • --amend → Letzten Commit nachträglich ändern
  • --force / -f → Aktion erzwingen (mit Vorsicht nutzen!)
  • --dry-run → Aktion simulieren ohne auszuführen
  • --verbose / -v → Mehr Details in der Ausgabe anzeigen
  • --all / -a → Alle Branches oder Dateien einbeziehen
  • --no-ff → Merge-Commit auch bei Fast-Forward
  • HEAD~1 → Stand des letzten Commits referenzieren
  • origin/main → Remote-Hauptbranch referenzieren
PROFI-TIPPS FÜR DEN ALLTAG
  • git add -p → Änderungen stückweise (interaktiv) merken
  • .gitignore anlegen → Dateien dauerhaft vom Tracking ausschließen
  • SSH-Key einrichten → Passwortfreies Pushen zu GitHub ermöglichen
  • git log --graph --all → Verlauf aller Branches als Baumstruktur
  • git bisect → Fehler durch Halbierungssuche finden
  • git alias → Eigene Kürzel für lange Befehle definieren
  • README.md schreiben → Projekt immer kurz dokumentieren
  • Branches früh löschen → Aufgeräumtes Repo ist leichter zu pflegen
